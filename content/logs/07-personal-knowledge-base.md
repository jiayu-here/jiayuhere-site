**项目：** 个人知识库与本地资料检索应用

**问题现象：** 只确认首页能够打开，无法证明登录、上传、解析、检索、详情、删除和退出功能真正可用。

**初步判断：** 问题不一定在页面本身，可能出现在会话 Cookie、文件挂载、解析流程或数据持久化环节。

**排查过程：** 先检查 Compose 配置和 Python 语法，再启动服务；使用保存 Cookie 的 HTTP 请求依次验证完整流程，并在容器重启后复查 SQLite 与上传资料。

**最终原因：** 单一首页健康检查覆盖范围不足，不能验证需要状态和持久化的用户流程。

**解决方法：** 建立带 Cookie 的端到端请求链，逐步验证登录、上传、提问、详情、删除与退出，并检查重启后的数据。

**经验总结：** Web 应用验收应覆盖真实用户路径和持久化，不应把“端口可访问”当作功能完成。
