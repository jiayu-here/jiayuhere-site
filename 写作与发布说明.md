# 写作与发布说明

网站已经使用 Markdown 作为文章、笔记和项目的内容源。平时只需修改 Markdown，不需要手写 HTML。

## 在哪里写

- 技术文章：`C:\Users\jiayu\Desktop\jiayuhere-site\content\articles\`
- 学习笔记：`C:\Users\jiayu\Desktop\jiayuhere-site\content\notes\`
- 项目详情：`C:\Users\jiayu\Desktop\jiayuhere-site\content\projects\`
- 可复制模板：`C:\Users\jiayu\Desktop\jiayuhere-site\content\templates\`

例如，写一篇新文章时，把 `content\templates\article-template.md` 复制到 `content\articles\`，重命名为 `05-my-article.md`，再填写标题、摘要和正文。

## 文件开头怎么写

每个 Markdown 文件都必须保留这一段：

```md
---
title: 文章标题
slug: unique-english-slug
description: 文章摘要
category: 分类
date: 2026-07-14
tags: [标签一, 标签二]
---
```

注意：

- `slug` 是页面网址的一部分，必须与其他内容不同。
- `slug` 建议只用英文小写、数字和连字符，例如 `freertos-task-scheduling`。
- 新文章文件名建议继续按顺序编号，例如 `05-freertos-task-scheduling.md`。
- 正文使用 `##` 和 `###` 写章节标题，也可以写列表、链接、行内代码和代码块。
- 项目文件还需要 `status` 和真实的公开 GitHub `repository`。私有仓库及本站仓库不会显示。
- 项目系统架构图使用 `architecture` 代码围栏，每行以 `模块 A -> 模块 B -> 模块 C` 描述一条数据流，网页会自动生成流程图。

## 本地先看效果

在 PowerShell 中运行：

```powershell
cd C:\Users\jiayu\Desktop\jiayuhere-site
npm run check
npm run build
npm run preview
```

然后访问 `http://localhost:8000`。文章会出现在“博客”，笔记会出现在“笔记”，项目会出现在“项目”页面；首页数字、站内搜索和站点地图也会同时更新。

## 让线上网站变化

确认内容后运行：

```powershell
git add content
git commit -m "Add new article"
git push origin main
```

推送到 `main` 后，GitHub Actions 会自动运行构建并提交生成后的网页，GitHub Pages 随后更新 `https://www.jiayuhere.com/`。不需要手动编辑 `blog`、`notes` 或 `projects` 中生成的 HTML。

如果只是修改现有文章，直接编辑原来的 `.md` 文件并重复提交、推送即可。删除某个 Markdown 文件后再次推送，对应的旧详情页也会在自动构建时清理。
