- 英文项目详情：`C:\Users\jiayu\Desktop\jiayuhere-site\content\en\projects\`
- 英文实验日志：`C:\Users\jiayu\Desktop\jiayuhere-site\content\en\logs\`

例如，写一篇新日志时，把 `content\templates\log-template.md` 复制到 `content\logs\`，按顺序重命名，再在 `content\en\logs\` 创建同名英文稿。两个文件的 `slug`、`date` 和 `projectSlug` 必须完全相同。

## 中英文如何同步

- 每个中文项目、文章、笔记和日志都必须有一个同名英文 Markdown 文件。
- 两种语言的 `slug` 必须相同；项目的 `repository` 也必须相同。
- 中英文日志的 `date` 和 `projectSlug` 必须相同；`projectSlug` 是项目页面网址中的目录名。
- 标题、摘要、分类、状态、标签和正文分别使用对应语言填写。
- `npm run build` 会自动生成对应的中文与 `/en/` 英文页面、搜索索引、RSS 和语言切换链接。
- 如果只新增中文文件而没有英文文件，构建会明确列出缺失文件并停止，线上不会出现跳转到空页面的语言按钮。

静态页面（首页、关于、工具箱、资源、联系和微信页）不是 Markdown 内容；以后修改这些页面的中文文字时，也要同步修改 `en\` 下的对应英文页面。静态网站不会在没有翻译服务和 API 的情况下自动生成可靠英文翻译。

## 文件开头怎么写

每个 Markdown 文件都必须保留这一段：

```md
---
title: 文章标题
slug: unique-english-slug
description: 文章摘要
category: 分类
date: 2026-07-14
tags: [标签一, 标签二]
---
```

注意：

- `slug` 是页面网址的一部分，必须与其他内容不同。
- `slug` 建议只用英文小写、数字和连字符，例如 `freertos-task-scheduling`。
- 新文章文件名建议继续按顺序编号，例如 `05-freertos-task-scheduling.md`。
- 正文使用 `##` 和 `###` 写章节标题，也可以写列表、链接、行内代码和代码块。
- 项目文件还需要 `status` 和真实的公开 GitHub `repository`。私有仓库及本站仓库不会显示。
- 日志文件使用 `projectSlug` 关联项目，例如项目地址为 `/projects/stm32-vision-robot-arm-sorter/`，就填写 `projectSlug: stm32-vision-robot-arm-sorter`。
- 项目系统架构图使用 `architecture` 代码围栏，每行以 `模块 A -> 模块 B -> 模块 C` 描述一条数据流，网页会自动生成流程图。

## 本地先看效果

在 PowerShell 中运行：

```powershell
cd C:\Users\jiayu\Desktop\jiayuhere-site
npm run check
npm run build
npm run preview
```

然后访问 `http://localhost:4173`。文章会出现在“博客”，笔记会出现在“笔记”，项目会出现在“项目”，日志会出现在“日志”页面；首页数字、站内搜索和站点地图也会同时更新。

## 让线上网站变化

确认内容后运行：

```powershell
git add content
git commit -m "Add new article"
git push origin main
```

推送到 `main` 后，GitHub Actions 会自动运行构建并提交生成后的网页，GitHub Pages 随后更新 `https://www.jiayuhere.com/`。不需要手动编辑 `blog`、`notes`、`projects` 或 `lab` 中生成的 HTML。

如果只是修改现有文章，直接编辑原来的 `.md` 文件并重复提交、推送即可。删除某个 Markdown 文件后再次推送，对应的旧详情页也会在自动构建时清理。
