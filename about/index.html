About Me

用工程验证想法，
用文档积累能力。

我关注通信、嵌入式、FPGA、数字信号处理和计算机系统。对我而言，学习不是收集名词，而是把一个问题从原理分析、可运行实现、仿真与调试，一直推进到他人能够复现和使用的交付物。

My Focus

我在做什么

目前以电子信息与计算机的交叉方向为主线，重点把通信原理、数字信号处理、FPGA RTL、STM32/FreeRTOS、Python 与工程环境连接成完整的实践链路。

我更在意一个系统是否有清晰的输入、输出、边界条件和验证证据，而不只是在页面或仿真中“看起来能跑”。因此，每个项目都会尽量保留源码、测试入口、工具版本、关键约束、结果说明和已知限制。

这个网站不是一次性的简历页，而是一份长期公开的学习档案：项目页说明做了什么、如何验证和哪些部分尚待完善；博客记录方法；实验日志保留排查过程与失败经验。

Working Style

我如何推进一个项目

把大问题拆成可独立判断的小阶段；每一阶段都尽量留下一个可以复查的结果，再进入下一阶段。

01SCOPE

先定义验收结果

明确目标板卡、输入输出、使用场景和完成判据。对 FPGA 项目，验收不止是生成文件，还包括仿真、约束与时序；对 Web 项目，则包括完整的用户流程。

02CORE

分层实现核心

把算法、时序控制、接口适配和展示层分开。这样出现问题时可以先定位在数据本身、模块边界还是板级连接，而不是在一个大文件中反复猜测。

03VERIFY

用证据推进调试

优先使用测试平台、波形、日志、串口抓包和 HTTP 请求验证真实行为。遇到异常时记录现象、假设、最小实验、根因和回归结果，而不是只保留最终版本。

04HANDOFF

完成文档与交付

整理源码、约束、脚本、运行说明、测试报告与已知限制。计划中的功能会明确标注为规划，而已经完成的部分才会给出可追溯的验证结论。

Technical Map

当前使用的技术

通信与 DSP：围绕调制解调、QPSK、误码率、定点实现、频率与采样关系建立理论和仿真的对应关系。

FPGA 与嵌入式：使用 Verilog、ModelSim、Quartus、时序约束、UART、I2C、AD/DA 和 DMA/FreeRTOS 等工具与接口，把逻辑从仿真推进到开发板。

软件与交付：使用 Python、MATLAB、SQLite、Docker、Git 和技术文档，承担可视化、检索、验证、环境复现与项目发布的工作。

Boundary

我如何说明能力边界

网站会区分“已完成”“规划中”和“后续改进”。已完成项目会列出实际使用的工具链、测试或编译证据；规划项目只记录目标架构、接口约定和验收条件，不提前把设计蓝图写成结果。

同样地，仿真通过、工程编译通过、时序通过和实物上板通过是不同层次的结论。我会尽量分别说明它们，避免用模糊描述掩盖尚未验证的部分。

这种做法也让我能持续回到旧项目：知道下一步从哪里补测试、补约束、补实测或补文档。
